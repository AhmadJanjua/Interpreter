 The interpreter being designed is a Tree-walk interpreter.

## Interpreter vs Compiler
Interpreter and compiler are not mutually exclusive ideas. A language can have elements from both.

Compiling is translating code from one source language to another, generally to a lower level. Generating bytecode, machine code, and transpiling are all forms of compiling.

Generally, when referring to a compiled language, it means a language that is compiled but does not execute the output. The user must execute the outputted code.

Interpreted languages take in source code and execute immediately, in a runtime environment.

This project was heavily supported by the book "Crafting Interpreters" by Robert Nystrom. I attempt to make a simple interpreter by following his guidelines, however instead of using Java, I use C++. This is to ensure that I implement the understanding rather than the logic from the book.
