## Interpreter vs Compiler
Interpreter and compiler are not mutually exclusive ideas. A language can have elements from both.

Compiling is translating code from one source language to another, generally to a lower level. Generating bytecode, machine code, and transpliling are all forms of compiling.

Generally, when referring to a compiled language, it means a language that is compiled but does not execute the output. The user must execute the outputted code.

Interpreted languages take in source code and execute immediately.
